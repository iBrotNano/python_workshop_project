## :triangular_ruler: Design

I want to refactor all code into classes. All datastructures should be model classes. All logic should be encapsulated into service classes.

I have to ways to define classes. For service classes I will use this way:

```python
class Person:
    def __init__(self, id: int, name: str, email: str = None):
        self._id = id
        self._name = name
        self._email = email

    @property  # lesen
    def id(self) -> int:
        return self._id

    @id.setter  # schreiben
    def id(self, value: int):
        self._id = value
```

- `__init__` with propertyies.
- "private" properties are named with an underscore.
- Getters and setters have decorators

Dataclasses are defined like this:

```python
rom dataclasses import dataclass
from datetime import date


@dataclass
class Kurs:
    id: int
    name: str
    start: date | None = None
    teacher: str = ""
    weeks: int = 4
```

All classes, methods and properties have type hints.

<details> 
    <summary>PR</summary>

A PR needs a title that lets the reviewer recognize which ticket it belongs to. The format is:

`#<issue number> <issue title>`

It helps the reviewer if you provide details about the development environment. Breaking changes in services can make it unclear which versions of services the reviewer should use for the review.

The versions can be set up from artefacts or via Git by using the correct branches.

If the exact version is not critical, it may be sufficient to simply use the image with latest.

Here is a template for a PR:

```markdown

## Notes

BREAKING CHANGE: Is this a breaking change?

Is there anything special to note? Perhaps deviations from the ticket or details that came up during development?

## Changes

- Fixed a typo
- Optimized code
- New feature XYZ

## Development environment

### Versions

| Application | Version |
| :---------- | ------: |
| Client      |   1.5.0 |
| Service     |   1.2.4 |
| KeyCloak    |  latest |
| Postgres    |  latest |

### Setup

Scripts or test data used? Ideally attach or link it.
```
</details>
